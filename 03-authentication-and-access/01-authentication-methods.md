# Understanding the different authentication methods in Entra ID

## Overview
I was a bit in doubt weather to create this lab since it's mostly going to consist of theory rather than a actual implementation. Still, I think it is extremely important to understand each of the different authentication methods provided by Entra ID. How are we supposed later to implement conditional access or secure privileged roles if we do not understand these options?

Authentication is the process of verifying a user's identity before the access is granted. Entra ID supports multiple authentication methods with different security levels. Understanding these methods is important because features such as conditional access, Privileged Identity Management, and Authentication strenghs rely on them when making access decisions.

Not every methods is equally secure, and there isn't one method that fits every situation. when designing an identity solution, it's important to think about both security and user experience/convinience. Some uders only need a secure and practical way to sign in, while highly privileged accounts should be protected with stronger authentication methods depending on the scenario and the level of access that needs to be protected.

## Objectives
- Explain the authentication methods in Entra ID
- Understand the purpose and the security characteristics of the most commonly used methods
- Configure selected authentication methods within the tenant
- Compare the different methods and understand when they are appropriate to use
- build the foundation for implementing Conditional Access policies and Privileged Identity Management in later labs
